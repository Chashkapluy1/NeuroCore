# Parallel Image Downloader 🚀

## 1. Описание проекта
Данный сервис решает классическую задачу I/O-bound: высококонкурентную загрузку X изображений с жестким лимитированием одновременных подключений до N.

Архитектура построена на принципах **Dependency Inversion**: мы используем абстрактный базовый класс (`BaseDownloader`), абстрагируя логику оркестрации от вызывающего кода. Это позволяет бесшовно переключаться между пулом потоков ОС (Threaded) и событийным циклом корутин (AsyncIO) без изменения внешнего контракта (`DownloadResult`).

## 2. Технологический стек
*   **Язык**: Python 3.11+
*   **Абстракция и Метаклассы**: встроенный модуль `abc`
*   **Сетевой слой**: `aiohttp` (для Async), `requests` (для Threaded)
*   **Дисковый I/O**: `aiofiles` (для неблокирующей записи)
*   **Надежность сети**: `tenacity` (для продвинутых повторных запросов)

## 3. Глубокий технический анализ подходов

### Coroutines (Asyncio & TaskGroup)
**Почему это лучший выбор для I/O-bound задач:**
В основе Async-подхода лежит концепция событийного цикла (Event Loop), работающего в одном системном потоке. При отправке HTTP-запроса поток не блокируется в ожидании ответа от сервера, а переключается на выполнение другой корутины.

*   *Легковесность:* Корутина — это просто объект состояния в памяти. Запуск 10,000 корутин потребует лишь десятки мегабайт RAM.
*   *TaskGroup:* Мы используем нововведение Python 3.11 — `asyncio.TaskGroup`. Это стандарт безопасного конкурентного выполнения (Structured Concurrency), который заменяет устаревший `asyncio.gather`. TaskGroup гарантирует, что если одна из дочерних задач падает с фатальным исключением, все остальные незавершенные задачи будут предсказуемо и безопасно отменены (cancelled), предотвращая утечки ресурсов.

### Multithreading (Пул потоков ОС)
**Почему это все еще работает в Python:**
Python печально известен своим GIL (Global Interpreter Lock), который запрещает одновременное выполнение байт-кода Python несколькими потоками. Однако, при ожидании сетевого ответа (I/O), стандартные библиотеки отпускают GIL. 
Таким образом, `ThreadPoolExecutor` позволяет пулу из N потоков находиться в одновременном ожидании загрузки разных файлов.

### Архитектурное сравнение
| Характеристика | AsyncIO | Пул Потоков ОС (Threading) |
| :--- | :--- | :--- |
| **Context Switching** | Минимальный (переключение стейтов в User Space) | Высокий накладной расход ядра ОС |
| **Потребление RAM** | Низкое (Кобайты на корутину) | Высокое (Каждый поток резервирует ~1-8 MB стека) |
| **Сложность кода** | Требует `async/await` везде («цветные функции») | Прост во внедрении в legacy-код |

## 4. Функции надежности (Aviation-Reliable)

Система спроектирована с учетом максимальной устойчивости к сбоям:

1.  **Smart Retries (Tenacity)**: Реализован алгоритм *Exponential Backoff*. При падении сокета система не долбит сервер спамом (DDoS), а удваивает время ожидания между попытками (1с, 2с, 4с, 8с) до 5 раз. Ошибки бизнес-логики (например, HTTP 404) не вызывают повторов.
2.  **Graceful Shutdown**: Внедрена защита от `SIGINT` (Ctrl+C). При попытке прервать скрипт во время загрузки 10-гигабайтного файла, система:
    *   Перехватывает сигнал с помощью подписки на события ОС.
    *   Активирует `threading.Event` или `asyncio.Event`.
    *   Мгновенно прерывает скачивание чанков дискового I/O.
    *   Очищает очереди задач и безопасно сохраняет частичные результаты в лог.
3.  **Структурированное логирование**: Применен стандарт раздельных Payload-потоков. Стандартный `INFO` идет в `stdout` терминала, а трассировки сложных ошибок и сбоев (`ERROR`) фиксируются в ротируемый файл `downloader_errors.log`.

## 5. Инструкция по запуску

### Быстрый старт
```bash
# 1. Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate  # Для Linux/Mac
# venv\Scripts\activate   # Для Windows

# 2. Установка зависимостей проекта
pip install -r requirements.txt

# 3. Базовый запуск (Использование как модуль)
```
```python
from async_downloader import AsyncDownloader

# Инициализация с лимитом N=10 и папкой ./images
downloader = AsyncDownloader(concurrency_limit=10, output_directory="./images")

# Запуск X URLs
urls = ["https://example.com/1.png", "https://example.com/2.png"]
results = downloader.download_all(urls)

print(f"Успешно: {sum(1 for r in results if r.success)}")
```

### Режим симуляции (Mock Mode)
Для нагрузочного тестирования оркестратора (поведения конкурентности без скраппинга интернета) передайте аргумент `mock=True`. Этот параметр симулирует I/O задержку ровно на 1 секунду.

## 6. Вывод
Реализованный паттерн является **Reference Architecture** для подобных задач. `AsyncDownloader` следует использовать как основное решение для высокой масштабируемости (X > 10,000 файлов). `ThreadedDownloader` предоставляется как совместимый fallback-вариант для систем с блокируемыми зависимостями, не поддерживающими AsyncIO. Оба варианта соответствуют Enterprise-стандартам безотказности.
